Scientific Summary: Autonomous Heuristic Discovery Framework

The framework presents an autonomous architecture designed to transition entity resolution from isolated, single-domain search operations into an integrated, agent-driven discovery loop. At its core, the system relies on Large Language Model (LLM) orchestration to synthesize heterogeneous data signals into a unified canonical person view, abandoning legacy silos.

The architecture enforces strict token efficiency through a "Skill-Based Routing" protocol (Yao et al., 2022). Rather than injecting the entire tool surface into the context window, the system uses a two-pass routing layer. The first pass evaluates a lightweight prompt containing one-line tool summaries, allowing the model to select required capabilities. The second pass injects only the selected, highly detailed tool schemas. This constraint reduces token consumption by 71% to 99% depending on the search domain, effectively preventing instruction drift during complex reasoning chains (Schick et al., 2023).

Heuristic iteration is managed through Learning-to-Rank (LTR) versioning (Liu, 2009). The historical trajectory of the LTR models demonstrates a critical architectural lesson: raw vendor data cannot be processed independently without a canonical identity join. Version 1 failed completely due to single-vendor bias, where row-level vendor prefixes prevented any cross-vendor overlap. The resolution (implemented in Version 3 and 4) requires resolving all vendor inputs against a centralized person-link structure before LTR processing occurs.

To safely deploy new entity resolution heuristics, the system implements a Four-Gate Waterfall evaluation process. This structure validates data bridges (such as professional network URLs, persistent company IDs, and stock tickers) against fill-rate thresholds. Heuristics that fall below confidence thresholds (such as fuzzy matching) are authorized only under a strictly labeled low-confidence tier, while maintaining a false merge rate below 5% (Christen, 2012).

Evaluation in the absence of external ground truth relies on internal consistency metrics: held-out vendor prediction, cross-vendor consensus, temporal plausibility using start and end dates, and stability under perturbation. Finally, the system models psycholinguistic interaction sequences using Markov Transition Models. By tracking transitions between search states, the architecture can detect "Stuck User" loops, characterized by high emotional tone and low analytic structure, triggering automated repair interventions before user abandonment.
