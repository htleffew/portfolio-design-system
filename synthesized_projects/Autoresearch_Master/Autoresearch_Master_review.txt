CHUNK 1 REVIEW: Core Architecture & Paradigm (Source Lines 9-54)
================================================================================

## 1.1 The Autonomous Analytical Loop
- **Paradigm Shift Identified:** The source describes a transition from the traditional human-driven hypothesis-testing cycle (design experiment -> engineer features -> tune hyperparameters -> evaluate -> repeat) to an LLM-delegated autonomous loop. [Source: Section 01, Lines 16-19]
- **Theoretical Foundation:** The pattern extends Automated Machine Learning (AutoML) architectures, citing Hutter et al. (2019). The LLM operates on a graph-based workflow: formulate hypothesis -> execute code -> evaluate results -> decide continue/terminate. [Source: Section 01, Line 18]
- **Core Risk Identified:** Granting an LLM "unconstrained mutation power" in a regulated enterprise environment "guarantees loop divergence and catastrophic provenance loss." The architecture must "formally leash the agent." [Source: Section 01, Line 19]

## 1.2 The Structural Divide (Fixed Harness vs. Mutation Budget)
- **Binary Dichotomy:** The architecture enforces a "strict physical dichotomy" splitting the loop's state space into two mutually exclusive regimes. [Source: Section 02, Lines 21-23]
  - **Fixed Harness:** Immutable enterprise constraints. Agent is "mathematically barred" from altering these.
  - **Mutation Budget:** Designated sandbox with finite computational allowance.
- **Key Design Principle:** The agent has freedom *only within* the mutation budget; it can never touch the harness.

## 1.3 Fixed Harness Components
- **Three Gate Types:** Pre-loop, mid-loop, and post-loop gates. [Source: Section 03, Lines 25-27]
- **Specific Gates Enumerated:**
  1. Enterprise Data Governance layer (forces PII redaction and classification)
  2. Role-Based Access Control (RBAC) layer (restricts agent to cohort-aggregate data, blocks individual-level retrieval)
  3. Responsible AI fairness gates
- **Enforcement Mechanism:** If the agent attempts to bypass the aggregate filter or disable the PII scrubber, "the Fixed Harness immediately terminates the execution." [Source: Section 03, Line 27]

## 1.4 Mutation Budget Mechanics
- **Four Mutation Domains:** hypothesis generation, feature engineering, model architecture/hyperparameters, evaluation metrics. [Source: Section 04, Line 30]
- **Cost-Deduction Model:** Every alteration deducts from the budget. Budget exhaustion triggers forced termination, preventing "infinite, hallucinated optimization spiral." [Source: Section 04, Line 31]
- **Interactive Simulator:** The case study includes a live simulator demonstrating:
  - Hyperparameter mutation (cost 10)
  - Feature logic mutation (cost 20)
  - PII bypass attempt (triggers immediate harness violation)
  - Budget visualization: bar turns yellow <40, red <20 [Source: Lines 33-38]

## 1.5 Divergence Monitoring (CUSUM)
- **Problem Addressed:** Even with bounded budget, LLM can "drift into hallucinated reasoning, optimizing for metrics that hold no real-world validity." [Source: Section 05, Lines 40-42]
- **Solution:** Cumulative Sum (CUSUM) control charts on loop metrics, citing Page (1954). [Source: Section 05, Line 42]
- **Trigger Condition:** Anomalous spike in model fit that "defies statistical plausibility" indicates agent engineered a data-leak rather than true signal.
- **Response Mechanism:** CUSUM trigger trips -> mutation-trace via experiment-tracking ledger -> automatic revert to last stable state.

## 1.6 Blocked-Restore Truth (Epistemic Provenance)
- **Core Principle:** Every mutation is written to a cutover-audit and restore-source ledger. [Source: Section 06, Lines 44-46]
- **Enforcement:** If the architecture "cannot find a pinned, authoritative source tree tracing back to the original unmodified dataset," the system "blocks the restore" and "explicitly refuses to accept the finding."
- **Design Intent:** Guarantees "undocumented 'black-box' conclusions are systematically purged from the pipeline."

## 1.7 The Application Boundary (Human Mandate)
- **Philosophical Purpose:** Preserving human judgment authority. [Source: Section 07, Lines 48-50]
- **Scope Limitation:** Agent output is bounded to "cohort-aggregate statistical supplements." It is "physically barred from evaluating individual records or emitting individual-level prescriptive decisions."
- **Division of Labor:** System generates exhaustively tested statistical evidence; the "licensed human business partner" makes all final application decisions.

## 3.1 The Karpathy Pattern (External Context)
- **Origin:** In March 2026, Andrej Karpathy released karpathy/autoresearch - an open-source tool automating the ML research cycle. [Source: Lines 178-180, Reference [1]]
- **Mechanism:** AI agent autonomously edits training code (e.g., train.py), runs fixed-time experiments (e.g., 5-minute) on single GPU, evaluates strict scalar metric (e.g., validation loss). Successful changes committed via Git; failures reverted.
- **Human Role Shift:** From writing code to writing constraints in plain-text program.md file. [Source: Reference [4]]
- **Broader Context:** Part of paradigm shift toward end-to-end autonomous discovery. Lu et al. (2024) "The AI Scientist" demonstrates industry shift from human-in-the-loop ML to fully automated hypothesis generation. [Source: Reference [2]]

## 3.2 Reproducibility and Provenance Justification
- **ML Reproducibility Crisis:** The blocked-restore posture is justified by modern ML reproducibility standards. Without exact end-to-end provenance traces (specific Git commit hashes, data lineage), ML pipelines cannot be trusted or reliably reproduced. [Source: Lines 185-186, Reference [3]]
- **Academic Backing:** Information Systems (2025) paper on capturing end-to-end provenance for ML pipelines validates the "full-tree proof" and "pinned authoritative revision" requirements. [Source: Reference [3]]
