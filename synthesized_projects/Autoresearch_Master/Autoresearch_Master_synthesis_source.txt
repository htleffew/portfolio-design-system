Title: Autonomous Research & Mutation Boundaries | Portfolio Research
Domain: Enterprise LLM Orchestration
Methodology: Agentic Graph-Based Workflows
Category: Methodological Case Study / Graph-Based Orchestration

Abstract: An architectural examination of bounding an unconstrained autonomous agent within a highly regulated enterprise environment. This case study details the structural dichotomy between the immutable Fixed Harness and the finite Mutation Budget.

SECTION 01: The Paradigm Shift - The Autonomous Analytical Loop
The traditional paradigm of applied machine learning relies on a human-driven hypothesis testing cycle. The data scientist designs an experiment, engineers the features, tunes the hyperparameters, evaluates the fit, and iteratively repeats the loop.
The "autoresearch" pattern shifts this paradigm by delegating the iterative hypothesis generation and experiment execution directly to a Large Language Model, extending foundational Automated Machine Learning (AutoML) architectures (Hutter et al., 2019). An autonomous analytical research loop runs on a graph-based workflow, where the LLM continuously formulates hypotheses, executes code to test those hypotheses, evaluates the results, and decides whether to continue or terminate the loop.
However, granting an LLM unconstrained mutation power over a research loop within a regulated enterprise environment guarantees loop divergence and catastrophic provenance loss. The architecture must formally leash the agent.

SECTION 02: The Dichotomy - The Structural Divide
To safely orchestrate an autonomous research agent, the architecture enforces a strict physical dichotomy over the loop's state space. The environment is split into two mutually exclusive regimes: the Fixed Harness and the Mutation Budget.
The Fixed Harness represents the immutable constraints of the enterprise. The agent is mathematically barred from altering these parameters. Conversely, the Mutation Budget represents the designated "sandbox" - the specific parameters the agent is explicitly authorized to alter in pursuit of a better model fit, bounded by a finite computational allowance.

SECTION 03: Immutable Boundaries - The Fixed Harness
The Fixed Harness ensures that no matter how aggressively the autonomous loop mutates its code, it cannot breach enterprise regulatory compliance. The architecture enforces pre-loop, mid-loop, and post-loop gates that the agent cannot circumvent.
These immutable gates include the Enterprise Data Governance layer (which forces PII redaction and classification), the Role-Based Access Control (RBAC) layer (which physically restricts the agent to cohort-aggregate data, blocking individual-level retrieval), and the Responsible AI fairness gates. If the agent writes a query that attempts to bypass the aggregate filter or disable the PII scrubber to gain more predictive signal, the Fixed Harness immediately terminates the execution.

SECTION 04: Bounded Exploration - The Mutation Budget
Within the protective boundary of the Fixed Harness, the agent is granted unconstrained exploration - up to a mathematical limit. The architecture allocates a Mutation Budget spanning four domains: hypothesis generation, feature engineering, model architecture/hyperparameters, and evaluation metrics.
Every time the agent decides to alter a feature or tune a hyperparameter, the cost is deducted from its budget. When the budget is exhausted, the loop forcefully terminates, ensuring the agent cannot run an infinite, hallucinated optimization spiral.

INTERACTIVE ELEMENT: Mutation Boundary Simulator
The page includes an interactive simulator with three buttons:
- "Mutate Hyperparameters (Cost: 10)" - valid mutation step, deducts 10 from budget
- "Mutate Feature Logic (Cost: 20)" - valid mutation step, deducts 20 from budget  
- "Disable PII Redaction Filter" - triggers FIXED HARNESS VIOLATION, immediate termination
The simulator starts with budget 100/100. Bar turns yellow below 40, red below 20. Budget exhaustion triggers SYSTEM HALT.

SECTION 05: Divergence Analytics - Divergence Monitoring (CUSUM)
Even with a bounded budget, an LLM can drift into hallucinated reasoning, optimizing for metrics that hold no real-world validity. The architecture must detect when the agent goes "off the rails."
To monitor this, the framework implements cumulative sum (CUSUM) control charts on loop metrics (Page, 1954). If the agent's feature mutations cause a sudden, anomalous spike in model fit that defies statistical plausibility (indicating the agent has engineered a data-leak rather than a true signal), the CUSUM trigger trips. The architecture executes a mutation-trace via the experiment-tracking ledger and automatically reverts the pipeline to the last stable state.

SECTION 06: Epistemic Provenance - Blocked-Restore Truth
The ultimate requirement of an autonomous research loop is traceability. A human must be able to verify exactly how a conclusion was reached. The architecture enforces an epistemic principle known as "Blocked-Restore Truth."
Every mutation the agent makes is written to a cutover-audit and restore-source ledger. If the autonomous loop presents a final research finding, but the architecture cannot find a pinned, authoritative source tree tracing back to the original unmodified dataset in the ledger, the system blocks the restore. It explicitly refuses to accept the finding. This blocked-restore default guarantees that undocumented "black-box" conclusions are systematically purged from the pipeline.

SECTION 07: The Human Mandate - The Application Boundary
The architectural rigor of the Fixed Harness and Blocked-Restore ledger serves a specific philosophical purpose: preserving human judgment authority.
The autonomous researcher is strictly an analytical engine. The architecture explicitly bounds its output to cohort-aggregate statistical supplements. It is physically barred from evaluating individual records or emitting individual-level prescriptive decisions. The system generates the rigorous, exhaustively tested statistical evidence, but the final application of that research remains exclusively the domain of the licensed human business partner.
