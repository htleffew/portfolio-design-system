### Stage 1: Extraction (Preprocessing)
- Read all provided raw source files. 
- Extract all text, code, and markdown. If the file is an image or PDF, perform OCR. If it's a Jupyter Notebook, extract only markdown and code blocks natively. 
- Collapse excessive blank lines, strip out ANSI escape codes, and normalize newlines.
- **Output:** Save this combined extracted text as `[ProjectName]_synthesis_source.txt`.

### Stage 2: Section Review (Analysis)
- **Persona:** Expert analyst.
- **Task:** Read `[ProjectName]_synthesis_source.txt` from disk. Process it in roughly 10k character chunks. Make detailed notes in a review format, including citations for where the information came from in the source material. Be concise but thorough.
- **Output:** Save as `[ProjectName]_review.txt`.

### Stage 3: Scientific Summary (Grounding)
- **Persona:** Rigorous scientific synthesizer.
- **Task:** **FLUSH CONTEXT.** Read ONLY `[ProjectName]_review.txt` from disk. Using the review notes, create a summary optimized for LLM understanding explaining all methods, perspectives, findings, processes, analyses, and results. It must be detailed enough for a scientific conference poster. 
- **CRITICAL:** Use your web search tools to find and include APA style citations to authoritative sources (peer-reviewed articles, arXiv, GitHub repos). Include full text citations and complete URLs.
- **Output:** Save as `[ProjectName]_summary.txt`.

### Stage 4: Article Creation (Karpathy/Olah Voice)
- **Persona:** Elite technical writer and educator.
- **Task:** **FLUSH CONTEXT.** Read `[ProjectName]_summary.txt`, `[ProjectName]_review.txt`, and `[ProjectName]_synthesis_source.txt` from disk. Collegial discussion, serving as an actual step-by-step tutorial. No hooks, em dashes, or formulaic narrative tropes. No building tension or manufacturing protagonists.
